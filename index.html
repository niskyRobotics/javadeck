Javadeck

A robot motion planning system for the FTC runtime.

View project on GitHub

About

Javadeck, created by FTC team 6460 "RoboWarriors", is a framework for quickly getting started with extremely intelligent and complex robot behaviors, on top of the US FIRST Java SDK. It is currently a work in progress.

Features

Realtime sensing and planning

The planner dynamically chooses goals based on the current state. A simple yet powerful API allows for dynamically controlling, enabling, and disabling goals in order to complete advanced tasks, where goals may be mutually-exclusive with, or pre-requisites for other goals.

Precise sensor fusion

A wide variety of sensors can be used for sensor fusion with a known field configuration. The dynamic integrators allow sensor input that is not necessarily a single location; sensors can "constrain" the robot to points, lines, curves, areas, or loci of any kind. This data can be used with the planner to make and execute decisions about movement.

At the moment the implementation is numerical (and already blazing-fast!), but there are plans for an algebraic API.

Advanced hardware control

The API for hardware control is designed using OO principles and is both efficient and organized. Filters such as gearing ratios and safety control allow interoperability between hardware and software, even with external influences such as possible failures or specific gearing ratios affecting effector speeds.

Upcoming:

Logging and error-handling

Thanks to the telemetry feature of the new FTC SDK, it is now possible to send realtime data back to the driver station. Integration with a set of joystick buttons will allow for human choice in handling abnormal error conditions during a match.
